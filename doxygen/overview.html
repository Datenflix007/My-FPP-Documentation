1. Project Discribtion

This project was commissioned as part of the "Fortgeschrittenes-Programmier-Praktikum" event at the Friedrich-Schiller-Universität Jena. The current product status was created in partnership by Felix Staacke and a fellow student. On Prof. Amme's behalf, they had to program a message server and a compatible client. These were to be able to communicate with each other within a local network. The two had to complete three milestones. First, they had to write a console application. In this, any number of clients can communicate in a message room on the server. Then, in the second milestone, they had to develop a GUI. Finally, several chat rooms and direct messages between users were implemented. Users can also be warned or blocked.

Following image shows the GUI of the Server.

The user interface has four areas. One area (top left) to create new chat rooms. One area (top right) to list the currently existing rooms (room name and description). One area (middle) lists all users who are online, offline, or blocked, filtered. New accounts can also be created in this area. In the fourth area (bottom), the log data are listed and filtered by session.
You can close the program via the menu ribbon under "Anwendung".
If you want to create a new Chat-Room, enter the roomname and the roomdiscribtion in the textfields. Than press "Neuen Raum erstellen". If your want to controll the chatrooms, than you can select one Chat-Room with a left mouse click. After clicking, a new frame will open. The messages are listed on the left. In the middle, the room name and the room description can be entered. The number of users who are currently using the selected chatroom is also displayed.

If you now click on a message, the frame on the right side expands. You can now also warn the user.

You can select a user in the middle area by left-clicking. Another window will then open. From this window you can warn, ban, or contact the user via direct message. If the user is warned, a message window opens for the user. If the user is banned, the socket connection is closed. The user can not then reconnect in the future using these login details.

In the lower area, you can show or hide the log entries of all sessions, the last session and the current session by left-clicking on the buttons.

When you run the client, a login window opens. There you enter a user name and password. Depending on the project order, a check is made to see whether the login data already exists. If so, the account is logged in. If not, a new account is created with the login data and logged in.

After logging in, the login window disappears and the client interface appears.

The user interface consists of three areas. The first (left) lists the option to create a new room (New Room), as well as all stored chat rooms. The second area (middle) lists all messages in a chat room. The third is on the right. This shows all users. If a room is selected, the users are filtered by "in this room", "in the other room", "offline". If no room is selected, the users are listed filtered by "Online" and "Offline".

5. Road to future

This project could become extended to an application for schools. If the server application will be extended, to generate simulated messages by Server. Both applications could be used to teach the students how to use social media plattforms. An possible experimental extension could be an implementation of Minigames. I.e. the Server could ask questions in one selected chat-room. The students should enter the right answer in the same chat.
